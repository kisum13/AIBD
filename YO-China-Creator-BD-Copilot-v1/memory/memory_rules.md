# Memory 维护规则

## 什么时候沉淀
以下情况必须沉淀：
1. 达人提出高频问题：报价、分成、合同、授权、付款、交付。
2. 出现新的谈判场景：混合模式、经纪人、多创作者资源、固定费过高。
3. 你觉得这句话“以后还会用”。
4. 这条回复最后推动了签约或有效推进。

## 怎么写 ID
格式建议：

```text
intent_language_序号
```

例子：
- `hybrid_model_en_001`
- `net_revenue_es_001`
- `content_workflow_ko_001`

## 怎么清洗
- 删除具体私人信息，例如手机号、邮箱、真实报价外的敏感信息。
- 保留业务逻辑和回复结构。
- 不要把“内部真实底价”写得太直白，避免未来误调用。
- 对收益预估必须标记为 potential / estimate / not guaranteed。

## 每条语料必须回答
1. 达人问了什么？
2. 我们为什么这样回？
3. 下一步要推进什么？
4. 是否有风险点？
